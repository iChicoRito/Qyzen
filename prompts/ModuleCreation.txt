Objective #1: Module Creation

    1). Create a new database table first name "tbl_modules" and must have these columns:

        - id
        - educator_id           (fk -> tbl_users.id)
        - subject_id            (fk -> tbl_subjects.id)
        - section_id            (fk -> tbl_sections.id)
        - module_id             (auto generated with this format e.g "MDL-123456789" with prefix of MDL and 1-9 random digiti)
        - module_code
        - time_limit
        - cheating_attempts
        - is_shuffle
        - is_active
        - start_date
        - end_date
        - start_time
        - end_time

    2). Now in this page (src\app\(educator)\educator\assessment\modules) is supposed for module Creation
        and replace all the tasks related for module creation. Since it's a static page and uses JSON,
        disregard that later once we need a real data from database.

    3). replace the "add-task-modal.tsx" into "add-modules-modal.tsx". The design of the modal should
        same with the other "add-****-modal.tsx" of this whole project.

    4). The add-modules-modal should have this fields:

        a). Module Code (dropdown and text field (meaning I can choose is it dropdown or I can type manually))
            - In the dropdown there's a pre-existing options
                - Quiz #1
                - Quiz #2
                - Quiz #3
                - Long Quiz
                - Written Exam
            
            - If manual, I can type those manually.

    5). Select Subject designated in the Section (Dropdown and can be multiple select in checkboxes)
        a). It shows all of the created subject on that section by that educator or
            who made that subject

        b). The tables related are:
            - tbl_academic_term
            - tbl_sections
            - tbl_sections_term
            - tbl_subjects

        c). So it shows the options like this

            PROGRAMMING FUNDAMENTALS    <- subject name
            BSIT1234 | PRELIM           <- this is a section and the term on that section

    6). Time Limit (text field)
        - this is where I can set a time limit of the modules

    7). Shuffle
        - This must be a switch or dropdown but I preferred switch

    8). Status (dropdown)
        - active or insactive, that's it

    9). Start and End Date (calendar)
        - I will choose the start and end date

    10). Start and End Time (time)
        - I will choose the start and end time

    11). For the validation, only 1 Module Code can be created in 1 subject/section.
