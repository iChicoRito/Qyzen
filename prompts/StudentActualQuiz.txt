Objective: Student Actual Assessment/Quiz taken on Module

    1). Now since the student can now see their modules, once they click the "Take Assessment" button, the student will be
        redirected to the actual assessment where the student will answer that choosen module.

    2). The actual taking of the quiz must be in the path (src\app\(student)\student\assessment\take-quiz). Right now, I don't have
        a page for that, so basically make that for me.

    3). That take quiz page must have these:

        The contents are from the "tbl_quizzes" and "tbl_modules" these are the related database tables, and you can still check
        the whole database in the path (database).

        a). It shows the modules information:
            - All the questions choices (if the quiz type is multiple choice) so it displays the radio button in 2x2
            - It shows all the questions and if the quiz type is identification, it just shows question and text input field.

        b). It reflects the timer based on the time limit, it shows as floating. To make it more immersive, if the time
            has more time, it uses the green badge based on the protocol, yellow for mid time and red for almost times out.

        c). Shuffles the questions and the choices based on the "is_shuffle".

        d). In the cheating attemps, I wanted that it shows warning whenever the student changes their tabs. The warning attempts will
            shows based on the "cheating_attempts". The warning shows as dialog. As u can see in most of the delete-confirmation-modal.tsx
            I wanted that kind of style and design of the Warning Dialog.

        e). When the student finish answering, there's a button "Submit" and before submitting, it shows a dialog for confirmation.

        f). Create a dedicated page for fallback that shows the student score, status, if failed or passed.

    4). Now for the scores, it will be stored in the new database table name "tbl_scores" with these columns

        - id
        - student_id        (fk -> tbl_users.id) who is the student answered the quiz
        - educator_id       (fk -> tbl_users.id) who is the educator of that student
        - module_id         (fk -> tbl_modules.id) what module is that quiz
        - subject_id        (fk -> tbl_subjects.id) what subject is that module
        - section_id        (fk -> tbl_sections.id)
        - score             (actuall score the student got)
        - student_answer    (It will stores here the students answer or what choice their clicking in realtime. It stores as JSON)
        - taken_at          (what time the student taken this quiz)
        - submitted_at      (what time the student submit the quiz)
        - ****              (add more timestamp for another information)

    5). Before you proceed to the execution, give a summary of the process, what is the plan and flow, what codes will be
        modified or added and delete (risky) and wait for the "Go!" command.
