Objective #1: Student Enrollment by Educator

    1). In the page src\app\(educator)\educator\enrollment, this page will be the enrollment page
        of the student on the subject. Since that page is for tasks and only static, modified that dedicated for
        enrollment.

    2). Create a new table in the database name "tbl_enrolled" with these columns:

        - id
        - student_id        FK->tbl_users.id      (the student)
        - educator_id       FK->tbl_users.id      (who's the educator who enrolled this student)
        - subject_id        FK->tbl_subjects.id   
        - is_active

        Check the path (database\schema\DatabaseSchema.sql) for better referencing.

    3). Now in the adding of the student, it will be a modal. Change the "add-task-modal" to "add-student-modal".
        The modal should contain these fields.

        a). Student Name (dropdown with checkboxes for multiple select and flexibility)
            - It shows all the student in the list
        
        b). Subject (dropdown with checkboxes for multiple select and flexibility)
            - It shows the subject name along with the section name in badge UI

    4). The modal layout and design should be unified to the other add-***-modal.tsx
        (e.g add-user-modal.tsx, add-section-modal.tsx and etc)

    5). The edit modal layout and design should be unified as well on the other view modal layout. The delete modal
        as well should same design to the others.

    6). Before you proceed to the execution, give a summary of the process, what codes will be modified or added and
        delete (risky) and wait for the "Go!" command.

    7). Since this static page uses JSON for premade data, it wont be used again once we already implemented the
        real data from supabase databbase.
